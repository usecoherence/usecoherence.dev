# Why Coherence

Software teams already make claims constantly: this change fixes the bug, this refactor is safe, this migration is complete.

Coherence is about making those claims bounded and reviewable.

The first version is deliberately small: specs, plans, runs, and evidence. Enough structure to reduce ambiguity without turning the workflow into a process tax.
