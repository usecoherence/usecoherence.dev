# Spec, AC, Plan, Run, Evidence

Coherence separates intent from execution and review.

- `spec` describes the intended change.
- `acceptance criteria` define what must be true.
- `plan` describes the checks that will be executed.
- `run` records what happened when the plan executed.
- `evidence` is the reviewable output attached to the claim.

The point is not ceremony. The point is to make software claims small, explicit, and auditable.
